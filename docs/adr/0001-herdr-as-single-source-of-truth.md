# herdrを状態の唯一の真実源とし、アプリ側の未読管理を持たない

「doneのエージェントをどのキャラがまだ確認されていないか」を、HerdWatch独自の未読フラグではなくherdrの状態（doneはpane閲覧まで維持され、閲覧で解除される）の鏡写しだけで表現すると決めた。二重管理はherdrサイドバーとキャラ表示の食い違いを生み、「どちらを信じるか」問題を持ち込むため。キャラタップ→paneフォーカスがherdr側の閲覧扱いを引き起こし、その状態変化イベントで自然にキャラも通常表示へ戻る。

## Consequences

- HerdWatchは状態を一切永続化しない（永続化するのはキャラクター割当のみ）。
- herdrが検出しきれない状態（hooks未導入時の精度）はHerdWatchでも同じ精度になる。改善はherdr側（hooks導入）で行う。
